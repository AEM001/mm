下面给出**“问题二”——在 0.7 T₁ 的时限内，把 期望总费用 （路途花费 + 期望罚款）最小化**的完整强化学习方案。  
所有位置用 **节点编号 0–99**（0 对应题目 1 号路口，99 对应 100 号）描述，已与 `limits_row` (10 × 9) / `limits_col` (9 × 10) 的编号排布一致。

## 1  额外数据与先验

| 项目 | 说明 | 记号 |
|------|------|------|
| **固定测速** | 所有限速 ≥ 90 km/h 的路段必有 1 台 | `fixed = (limit ≥ 90)` |
| **移动测速** | 一共 20 台，随机分布到全部边（记总边数 `E = 10×9 + 9×10 = 171`） | 每条边有移动雷达的概率 `p_m = 20 / 171 ≈ 0.117` |

单雷达探测率 `p₁`  
| 超速幅度 r | ≤20 % | 20–50 % | 50–70 % | 70 % |
|------------|-------|---------|---------|------|
| **p₁** | 0 | 0.70 | 0.90 | 0.99 |

## 2  每条边的 **期望费用** 公式

设  
- 路段限速 `V_lim`  
- 选定 **超速比例** `r ∈ {0, 0.2, 0.5, 0.7}`（对应 0 %、20 %、50 %、70 %）  
- 实际车速 `v = V_lim × (1+r)`（上限 1.7 V_lim）

### 2.1 旅途费用（同问题一公式，速度换成 *v*）
\[
\begin{aligned}
t &= \frac{50}{v}\quad(\text{h})\\
q &= \Bigl(0.0625\,v+1.875\Bigr)\times0.5\quad(\text{L}) \\
c_\text{travel} &= 20t + 7.76\,q + \bigl[25\;\text{if}\;V_{lim}=120\bigr]
\end{aligned}
\]

### 2.2 罚款的 **期望值**

1. **罚金面额 F(V_lim,r)** ——按题目《交通法规》对照表确定  
2. **单雷达探测率** `p₁(r)`（见上表）  
3. **边上雷达数量分布**  
   - `k = 1` (只有固定) 概率 `1-p_m`  
   - `k = 2` (固定 + 移动) 概率 `p_m`  
   - 对限速\<90 的路段 `k=0` 或 `1`（有无移动）  
4. **被查概率**  
   \[
   P_{\det} = 
   \begin{cases}
   p_m\,(1-(1-p₁)^1), & \text{无固定雷达}\\[4pt]
   (1-p_m)\,(1-(1-p₁)^1) + p_m\,(1-(1-p₁)^2), & \text{有固定雷达}
   \end{cases}
   \]
5. **期望罚款** `c_fine = P_det × F(V_lim,r)`

### 2.3 **总期望费用**
\[
c_\text{edge}(V_\text{lim},r) \;=\;
c_\text{travel} \;+\; c_\text{fine}
\]

（出于效率，可把 4 种 *r* × 5 种限速预先离线算好，生成 20 张 lookup 表。）

## 3  强化学习建模

### 3.1 状态 **S**
```
s = (node_id,  time_bucket)
```
- `node_id` ∈ 0…99  
- `time_bucket` = 剩余可用时间离散化（例如把 0 – 0.7 T₁ 线性划成 10 桶）。  
  > 这样智能体可显式感知“还能用多少时间”并动态决定要不要再加速。

### 3.2 动作 **A**
```
a = (dir, r_idx)
```
- `dir` ∈ {0,1,2,3}  四方向  
- `r_idx` ∈ {0,1,2,3}  对应超速比例 r∈{0,0.2,0.5,0.7}  
（若 `v > 1.7 V_lim` 或该方向无路 ⇒ 非法动作，奖励 –1000）

### 3.3 转移 & 奖励
1. 查表拿 `c_edge`，即时奖励  
   \[
   r = -\,c_\text{edge}
   \]
2. 更新时间：`remaining_time -= t`  
3. 终止：
   - 成功：到达 99 且 `remaining_time ≥ 0`  
   - 失败：`remaining_time < 0` 或步数 > 150  
     → 给予一次性大罚 `-10 000`（逼智能体满足时限）

### 3.4 目标
> **最小化期望费用** 且隐式满足时限 —— 在上面的 reward 设计下，等价于 **最大化累积奖励**。

## 4  算法推荐

| 规模 | 算法 | 备注 |
|------|------|------|
| `|S|≈100×10`, `|A|≤16` | **表格 Q-Learning with constraint** | 极小 Q-table (< 16 000)，训练快；通过“大罚”让策略学会守时。 |
| 若想显式约束 | **Lagrangian Q-Learning** | 维护 λ，更新规则：<br>λ←[λ+β( expected_time – 0.7T₁ )]⁺ |

> 先用简单“大罚”法跑通，再用拉格朗日法做收敛性微调。

## 5  训练步骤

1. **T₁ 已知**（问题一输出），计算 `deadline = 0.7 T₁`  
2. 初始化 Q(s,a)=0，ε=1  
3. **loop episode**  
   - `state ← (0, deadline)`  
   - while not done:  
     - 取合法动作集合；ε-贪婪采样  
     - `reward, next_state` ← `step(a)`（按 §3.3）  
     - Q-update：`Q[s,a] ← Q + α(r + γ·max_a'Q[s',a'] - Q[s,a])`  
     - `state ← next_state`  
   - ε线性衰减到 0.05  
4. 监控最近 200 episode 的平均 **期望费用**，收敛后停止

---

## 6  路线三与超速方案提取

1. **贪心解码**：从 `(0, deadline)` 起，每步取 `argmax_a Q[s]`  
2. 记录序列  
   ```
   Route3 : node0 → node… → node99
   SpeedPlan : [(node_from,node_to,r%), ...]
   ```  
3. 同时累加  
   - 实际用时 T₃（应 ≤ 0.7 T₁）  
   - 旅途费用  
   - 期望罚款  
   - **C₂ = 旅途费用 + 期望罚款** （输出保留两位小数）

---

## 7  快速 sanity check

- **T₃ / 0.7T₁** ≈ 0.9–1.0 → 若 >1 则 Q 未收敛  
- **高速路段**：大概率会降速到 90 km/h（r=–25 %）或原速 120，而很少选 r=0.7；反之限速 60 的路段常选 r=0.2 左右  
- 与 **约束最短路算法**（Constrained Dijkstra）对比路线 & 费用，应基本一致

### 小结

1. **期望费用** = 旅途三项成本 + 期望罚金（考虑雷达数量与探测概率）  
2. **动作**：方向 + 超速档（0/20/50/70 %）  
3. **时限约束** 用剩余时间入状态并对“超时”给予巨额负奖励；或用拉格朗日乘子版 Q-Learning 精细调  
4. 训练收敛后，按照 Q-table 贪心抽取 **路线三 + 每段超速比例**，计算最小期望费用 **C₂**。

## 1. 把题目转成一个带时限与罚款的 MDP

### 1.1  随机性来源  
1. **移动测速雷达位置**：每个 episode 开始时，20 台移动雷达随机落在 171 条边之一（独立同分布）。  
2. **被抓概率**：给定超速幅度 r，单雷达命中率 *p₁(r)* 已知 → 多雷达命中率随 k 变化。  
> 这两重随机使罚款成为期望量，最自然的决策框架是 **随机 MDP** 而非确定性最短路。

### 1.2  状态 S  
```
S = (node,   t_left_discrete)
```
- `node ∈ {0…99}`  
- `t_left_discrete` 把剩余时间 **[0, 0.7T₁]** 划成 *B* 个桶  
  - 取 *B = 15* 时：状态总量 = 100 × 15 = 1500  
> 离散化越细，可行性越好；过细会让 Q-table 稳定变慢。15–20 个桶在经验上收敛速度与精度折衷最佳。

### 1.3  动作 A  
```
A = (dir, r_idx)
```
- `dir ∈ {0,1,2,3}` 四方向  
- `r_idx ∈ {0,1,2,3}` → r∈{0%, 20%, 50%, 70%}  
  - 0% = 不超速；70% 为题目上限  
> 行为空间 |A| ≤ 16，小而可控 → **表格 RL 足够**。

### 1.4  过渡函数  
1. 环境先根据 `dir` 判断是否存在道路 & 限速 `V_lim`  
2. 若非法：状态不变，reward = –1000  
3. 若合法：  
   - 生成本 episode 中该边的 `k = 固定(0/1)+移动(0/1)`  
   - 计算：  
     - 实际速度 **v = V_lim × (1+r)**  
     - 行驶时间 **t = 50/v**  
     - 旅途花费 **c_travel**（三项）  
     - **P_det(k,r)** → 期望罚款 **c_fine = P_det × F(V_lim,r)**  
   - `reward = – (c_travel + c_fine)`  
   - `t_left ← t_left – t`；若 `<0` 则环境加一次性 –10000 罚分并 `done=True`  

### 1.5  目标  
最大化  
\[
\mathbb{E}\!\left[\sum_{t=0}^{T} r_t \right]
\]  
即可 **最小化期望总费用** 且自动规避超时。

## 2. 为什么「期望罚款」就够用？

- 雷达分布对司机无可观测，且司机只能事后知道是否被抓。  
- 因此最优策略只能对“概率”而非具体分布做决策。  
- 把罚款铸造成 **期望值** 就把高维不确定性压缩进单一标量 `c_fine`，既保留了随机性影响，又不引爆状态维度。  
> 若要进一步“保守”或“风险厌恶”，可把期望替换成 **CVaR** 奖励或引入 **Distributional RL**；下文仍以期望为主线。

---

## 3. 解决方法 1 —— **约束松弛 + 大罚** 的表格 Q-Learning

|          | 说明 |
|----------|------|
| **思路** | 把「不能超时」改成「超时一次性 –10000 分」。<br>只要 –10000 明显劣于任何合法路线花费（上千元量级），最优策略会自然满足时限。 |
| **算法** | 普通 Q-Learning：<br>· Q(s,a) 大小 ≤ 1500×16 ≈ 2.4 万<br>· 1e5–2e5 步即可收敛 |
| **优点** | 代码改动最小；与前两题同一框架，便于复用。 |
| **缺点** | 大罚值不好调：过大导致梯度爆炸、过小可能仍偶尔超时。 |

> **经验做法**：先用 –1e4 训练 3–4 k episode 观察超时频次；若仍 >0.1 %，再放大到 –2e4。

---

## 4. 解决方法 2 —— **拉格朗日乘子 (Primal-Dual) Q-Learning**

> 把“期望时间 ≤ 0.7T₁”当硬约束，在线学习 λ。

### 4.1  优化目标  
\[
\max_\pi \min_{\lambda\ge0}
\; \mathbb{E}_\pi\!\bigl[R\bigr] \;-\; \lambda\left(\mathbb{E}_\pi\!\bigl[T\bigr]-0.7T₁\right)
\]

### 4.2  算法流程  
1. **双表格**：  
   - Q(s,a) 学 **Lagrangian reward** `r_λ = reward - λ·t`  
   - λ scalar，初始化 0  
2. **每 N 步** 更新 λ  
   \[
   \lambda \leftarrow \Bigl[\lambda + \beta\bigl(\hat{T}-0.7T₁\bigr)\Bigr]^+
   \]
   `β` 可取 0.05  
3. 收敛判据：  
   - λ 稳定振荡在某个值附近  
   - 统计表观平均行驶时间 ≈ 0.7T₁

| 优劣 | 评价 |
|------|------|
| **优** | 理论保证：最终满足时间约束且费用近最优。<br>罚值自适应，无需手调 –10000。 |
| **劣** | 实现略复杂：要记录每集总时间估计 `\hat T`。<br>λ 太大时 early training 难收敛，需要稀疏更新或 warm-up。 |

---

## 5. 解决方法 3 —— **资源受限最短路径 (Dynamic Programming)**

> 状态 = (node, 累计时间)，时间离散同上；直接跑 DP 而不是 RL。

### 流程  
1. 定义 `cost(node, t_used)` = 已知最小期望费用到达该状态  
2. 按 **时间升序** 松弛所有边 (`dir, r_idx`)：  
   \[
   cost(n', t'+t) = \min\bigl\{cost(n', t'+t),\;cost(n,t') + c_\text{edge}\bigr\}
   \]
3. 最终在 `node=99` 所有 `t_used ≤ 0.7T₁` 中取最小费用

| 评价 | |
|------|--|
| **优** | 无采样误差；一次跑完即全局最优 |
| **劣** | 时间维度取 10–20 桶仍 OK（171×20×4×4 ≈ 5.5 k 更新）；<br>但若想把时间当 **分钟级离散** → 状态爆炸。<br>且不易兼容后期加入「路段拥堵随机」等更复杂动态。 |

### 结论

- **建模关键**：把超速→被抓→罚金转成期望值，剩余时间离散进状态。  
- **算法可行性**：  
  - **小规模**：最简表格 Q-Learning（或 DP）即可；  
  - **硬约束**：用拉格朗日 Q-Learning 拿到理论保证。  
- **输出**：训练后贪婪解码得到 **路线三 + 每段超速比例**，并计算期望最小费用 **C₂**。  

按上述任一方案实现，都能在几分钟内得到满足「0.7 T₁ 以内、费用期望最小」的策略，且框架可平滑扩展到更高维场景。
